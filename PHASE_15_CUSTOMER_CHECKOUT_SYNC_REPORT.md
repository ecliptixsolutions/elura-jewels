## Implemented

- Checkout is centralized through `StoreContext.checkout`.

- Newsletter Shopify mutation now avoids local-clock `consentUpdatedAt` failures.

- Newsletter network/proxy errors now show a user-friendly message instead of raw `502`, `AggregateError`, or proxy text.

## Root Causes Fixed

### Checkout Guest Orders

Root cause: checkout could be created without Firebase auth and without forcing Firebase email into Shopify buyer identity.

Fix: all checkout entry points now pass through the centralized protected checkout function.

### Cart Not Clearing After Purchase

Root cause: Shopify checkout happens off-site, and the app did not detect a completed order when the customer returned.

Fix: the app stores the checkout timestamp, checks Shopify order history on return/focus/pageshow, and clears cart state only when a matching new Shopify order is found.

### Newsletter 502 / ECONNREFUSED

Root cause: Vite proxied `/api` to `localhost:5000`, but `npm run dev` started only Vite.

Fix: `npm run dev` now starts both `backend/server.js` and Vite through `scripts/dev-with-api.mjs`.

### Newsletter Shopify Mutation Failure

Root cause: `consentUpdatedAt` was generated by the local machine clock and Shopify rejected it as future-dated.

Fix: removed `consentUpdatedAt` and let Shopify set the consent timestamp.

### Duplicate Newsletter Failure

Root cause: existing customers must update marketing consent through `customerEmailMarketingConsentUpdate`, not `customerUpdate`.

Fix: duplicate/existing customers now update tags and marketing consent with the correct mutations.
